<h1 style="text-align:center;font-size:24px;margin-bottom:24px;color:#333">为什么开发这个软件</h1>

<p>我干测绘和无人机这行有几年了，格式转换一直是工作中最繁琐的环节——不是因为它难，而是因为重复性太高，每次都在消耗宝贵的工作时间。</p>

<p>举个例子。你用CAD或者GIS，导出个 KML 范围线，想拿去 UOM 申报空域。结果呢？UOM 根本不认你那个 KML。不是格式不对，是里面的结构、坐标系、甚至文件大小它都有自己的要求。你得手动重绘一遍，或者找个中间工具转换半天。画一个面域可能也就三五分钟，但攒多了呢？一个项目几十块地，光转格式就要花费半个下午。</p>

<p>还有司空 2 那个 20MB 的限制。大面积测区导出的 KML 动不动就几十上百兆，直接导入就报错。你得手动把一个大面域切成七八块，每块还不能乱切——边界要对齐，范围不能有缝隙，还得保证每块都在 20MB 以下。这活做一次还行，做多次就让人头疼了。</p>

<p>图斑调查也很麻烦。领导下发一个 KML，里面套了三十块图斑，你要一块一块分离出去让不同的人去核查。手动操作的话就是打开文件，复制粘贴，另存，重命名，关掉，再打开下一个。我认识的同行有人专门为这个写了 Excel 表格来跟踪进度，因为太容易遗漏了。</p>

<p>这几件事有一个共同点：都是重复性的基础工作。但偏偏你不做又不行，数据到不了下一个环节，后续的空域规划和航线设计都无法进行。</p>

<p>所以我开发了这个工具。初衷很简单：第一，格式转换这种重复劳动应该由工具来完成。第二，工具得够简洁够高效，别搞一堆菜单层级让人点来点去。我设了三个场景——标准模式、司空 2 模式、UOM 模式——每个都只解决问题，不添麻烦。操作简单直观，不需要看说明也能上手，这就是我对一个好工具的理解。</p>

<p>希望这个工具能帮到更多同行，让大家把时间花在更有价值的事情上。</p>
